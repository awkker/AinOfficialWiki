# 正文页面包屑自适应折叠设计

## 背景

当前正文页顶部面包屑存在三个稳定性问题：

- 面包屑外壳宽度与正文容器宽度分别维护，只有少数断点能左右对齐。
- 路径折叠只按固定数量处理，没有根据容器宽度、方向或 TOC/aside 状态调整。
- 当前页标题仅靠固定 `max-width` 截断，窄屏和竖屏下会出现两行换行或整体过长。

## 目标

为正文页面包屑实现容器驱动的响应式布局：

- 与正文内容区保持稳定对齐，不依赖脆弱的固定偏移。
- 根据可用宽度自动切换折叠模式。
- 横屏优先保留首页和末尾 2 到 3 项。
- 竖屏优先保留首页和当前页。
- 当前页标题始终单行，必要时省略，并可通过 Popover 查看完整文本。
- 折叠出来的路径仍可通过 Popover 正确访问。

## 参考模式

参考 HeroUI、Nuxt UI、shadcn/ui 的通用做法，采用以下组合：

- 面包屑主体使用单行 flex 容器，禁止换行。
- 中间层级折叠为一个省略触发器。
- 省略触发器通过 Popover 或 Dropdown 展示隐藏路径。
- 当前页项目单独设置弹性宽度预算和文本省略。

## 实现思路

### 1. 容器感知折叠

在 `Breadcrumbs.vue` 中通过 `ResizeObserver` 观察导航容器宽度，并结合 `matchMedia('(orientation: portrait)')` 推导折叠模式：

- `full`：宽度充足时全部显示。
- `tail-2`：保留首页、省略中间、保留末尾两项和当前页。
- `tail-1`：保留首页、省略中间、保留倒数第二项和当前页。
- `root-current`：仅保留首页、省略中间、当前页。

### 2. 数据层输出可渲染结构

在 `breadcrumbs.ts` 中把原来的固定折叠函数改成接受折叠模式的纯函数，确保：

- 少层级时直接返回全部项目。
- 需要折叠时只隐藏中间项目，首页和末尾关键路径稳定保留。
- 当前页始终作为最后一个可见项目输出。

### 3. 布局与宽度预算

在样式层做以下约束：

- 面包屑整体和列表都强制单行。
- 所有非当前页项目使用 `flex: 0 0 auto`，防止被压缩得不可读。
- 当前页项目使用 `flex: 1 1 auto; min-width: 0;`，将剩余宽度全部交给标题。
- 标题使用 `overflow: hidden; white-space: nowrap; text-overflow: ellipsis;`。
- 面包屑外壳改为直接继承正文上下文宽度，移除脆弱的固定平移。

## 验证标准

- 关闭或显示 TOC/aside 时，面包屑与正文左右边界保持一致。
- 窄屏与竖屏下不出现两行文字。
- 长标题仅最后一项截断，Popover 能展示完整标题。
- 横屏宽度增大时，自动恢复更多尾部层级。
- 构建通过，无新增主题编译错误。

## 不做的事情

- 不引入额外 UI 库。
- 不改动面包屑数据来源和站点导航生成逻辑。
- 不增加新的全局断点体系，只围绕当前组件做容器驱动适配。
